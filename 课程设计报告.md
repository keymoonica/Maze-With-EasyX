# **1** **绪论**

迷宫问题是图形学、图论和数据结构等领域中广为人知的一个经典问题。战略决策、机器人路径规划等许多智能问题都可以转化为寻找迷宫最优路径问题^[1]^。其核心在于如何通过非线性结构（如图、网格）抽象空间关系，并应用线性结构（队列、栈）实现高效搜索。而传统的迷宫研究多聚焦于基于传统搜索算法深度优先搜索（DFS）和宽度优先搜索（BFS）^[2]^的算法效率的理论分析，局限于命令行输出或静态图示，无法动态展示算法决策、路径回溯及实时移动轨迹，缺乏直观的可视化展示。

为突破此局限，本研究设计并实现基于EasyX图形库，在原生C/C++环境下集成以下功能的系统：

1. 迷宫生成：支持固定迷宫模板与基于Prim算法下的随机迷宫生成，确保了迷宫结构的多样性。
2. 路径求解：支持BFS与DFS两种路径寻找算法，并提供了动态展示搜索过程、路径节点扩展和回溯机制。
3. 可视化交互：可以实时绘制迷宫状态、动态显示算法的实现过程，记录并可视化路径的移动轨迹。

研究范围涵盖迷宫生成算法、路径搜索算法的实现优化，以及EasyX图形库的高效可视化技术。用户可通过GUI按钮触发不同的操作（生成迷宫、选择算法、启动测试），系统可以实时显示迷宫状态、算法搜索的提示路径、移动代理的实时路径、移动轨迹日志。

通过此系统的运用不仅将抽象算法转化为直观动态过程，提高用户对搜索策略的理解，还为复杂路径规划算法，如机器人导航、小鼠迷宫等提供可视化调试工具。并且由于模块化编写，该系统框架支持集成更多算法，为接下来的算法可视化研究具有一定建设性。

## **3.2路径搜索功能**

### 3.2.1 广度优先搜索（BFS）

使用`find_path_bfs`函数实现BFS算法。该算法使用队列来存储待探索的节点，从入口出发，离开入口后依次访问与当前位置邻接的单元格，然后分别从这些相邻单元格出发依次访问它们的邻接格，并使“先被访问的单元格的邻接格‘先于’后被访问的单元格的邻接格”被访问，直至访问到迷宫出口，则找到了迷宫问题的最优解，即迷宫最短路径[1]。在搜索过程中，使用`visited`数组记录已访问的节点，使用`parent`数组记录每个节点的父节点，以便最后回溯路径。

### 3.2.2 深度优先搜索（DFS）

`find_path_dfs`函数实现了DFS算法。该算法使用栈来存储待探索的节点，从入口出发，离开入口后依次访问与当前位置邻接的单元格，然后分别从这些相邻单元格出发依次访问它们的邻接格，并使“先被访问的单元格的邻接格‘先于’后被访问的单元格的邻接格”被访问，直至访问到迷宫出口，则找到了迷宫问题的最优解，即迷宫最短路径[1]。同样，使用visited数组和parent数组来记录访问状态和父节点信息。

## **3.3 可视化展示功能**

可视化展示功能主要包含三个方面。迷宫绘制由`draw_maze`函数实现，该函数依据迷宫的单元格类型（墙、路径、起点、终点），利用EasyX图形库把对应的图片绘制到屏幕上，以此达成迷宫的可视化展示；路径绘制通过`draw_path`函数完成，它会每隔100毫秒更新当前位置，同时绘制提示路径和当前位置的图片；移动轨迹绘制由`draw_movement_trace`函数负责，其将移动轨迹以文本形式显示在屏幕上，便于用户查看。

## **4.3 关键算法设计**

### 4.3.1 Prim迷宫生成算法

Prim算法是一种用于生成最小生成树(MST)的经典算法[4]，常用于图论中。在迷宫生成的场景中，我们可以将其变种应用于生成随机、自然的迷宫结构。与传统Prim算法不同，迷宫生成中的Prim算法不是寻找最小生成树，而是通过随机选择和打通路径来创建连通的迷宫。

最开始的迷宫可以看作是所有墙都是关闭的，迷宫生成的过程则是使用`prim`遍历整个迷宫的过程中把墙打开。具体步骤如下：

1. 初始化所有的单元格都为墙；
2. 随机选择一个单元格为起点将其标记为路径；
3. 将起点周围的墙加入墙列表；
4. 重复以下步骤直至墙列表为空：随机选择一堵墙，如果这堵墙分隔了一个已访问的单元格和未访问的单元格，则打通这堵墙，并将未访问的单元格标记为已访问，将新的单元格周围的墙加入墙列表，否则移除这堵墙。

数据结构协同工作的流程为：首先初始化`grid`数组全为WALL，随机选择起点，如(3,3)并标记为CELL_PATH，将起点周围的墙(2,3)、(4,3)、(3,2)、(3,4)加入墙列表；接着处理第一堵随机选中的墙(2,3)，检查发现其左侧(1,3)为`CELL_WALL`、右侧(3,3)为`CELL_PATH`，于是打通该墙，将(2,3)和(1,3)标记为`CELL_PATH`，并将(1,3)周围的墙(0,3)、(2,3)、(1,2)、(1,4)添加到墙列表；随后处理下一堵随机选中的墙(4,3)，其左侧(3,3)为`CELL_PATH`、右侧(5,3)为`CELL_WALL`，打通后标记(4,3)和(5,3)为`CELL_PATH`，再将(5,3)周围的墙(4,3)、(6,3)、(5,2)、(5,4)加入墙列表，如此重复直至墙列表为空。

### 4.3.2 广度优先搜索（BFS）算法

广度优先搜索是一种相当常用的图论算法,其特点是:每次搜索指定点,并将其所有未访问过的邻近节点加入搜索队列,循环搜索过程直到队列为空[5]。在迷宫问题中，BFS从起点开始逐层扩展，直到找到终点或遍历完所有可达节点。

具体算法流程为，先将起点加入队列并标记为已访问，然后循环处理队列中的每个节点，在每一次处理时先检查当前节点是否为终点，若是则通过父节点回溯构建路径并返回，若不是则将所有相邻且可达的节点加入队列，同时标记这些相邻节点为已访问并记录其父节点，若队列为空时仍未找到终点，则表明迷宫中不存在从起点到终点的路径。

BFS算法在数据结构上使用队列实现逐层扩展以确保按层次遍历节点，借助二维数组记录每个节点的访问状态避免重复访问，同时利用二维数组记录各节点的父节点以便后续回溯路径；

### 4.3.3 深度优先搜索（DFS）算法

深度优先搜索方法从初始节点开始,顺序扩展生成下一级各子节点,放在OPEN表中已有的节点前面(实现后生成的子节点先扩展),然后从OPEN表中提取最前的一个节点检查是否是目标节点,否则再扩展,再重复上述操作[5]。在迷宫问题中，DFS从起点开始沿着一条路径尽可能深地探索，直到无法继续或找到终点，然后回溯到上一个节点继续探索其他路径。

具体算法流程为：从起点开始并标记为已访问，随后递归探索当前节点的所有相邻节点，每次探索时先检查当前节点是否为终点，若是则通过父节点回溯构建路径并返回，若不是则选择一个未访问的相邻节点继续探索，回溯时若发现所有相邻节点都已访问则返回上一层，若所有路径都被探索完仍未找到终点，则表明迷宫中不存在从起点到终点的路径。

DFS算法在数据结构上通过使用递归调用栈来实现路径探索，借助二维数组记录每个节点的访问状态以避免重复访问，同时利用二维数组记录各节点的父节点以便后续回溯路径。

与BFS在迷宫算法相比，DFS不保证最短路径，而BFS能确保找到最短路径；内存使用上，DFS因利用递归调用栈，内存消耗较低，BFS则因需维护队列存储逐层扩展的节点，内存使用较高[6]；遍历方式上，DFS沿一条路径深入探索，BFS则是逐层扩展；适用场景中，DFS适合探索未知区域，BFS更适用于需要最短路径的场景；路径特点上，DFS生成的路径可能较长且曲折，BFS的路径则最短且直接。

### 5.2.2 随机迷宫生成模块（Prim算法）

```C
void generate_random_maze(Maze* maze) {

  srand(static_cast<unsigned int>(time(NULL)));  // 显式类型转换 // 初始化随机数种子

  // 初始化全为墙的迷宫

  for (int row = 0; row < ROWS; row++) {

    for (int col = 0; col < COLS; col++) {

   maze->grid[row][col] = CELL_WALL;  

    }

  }

  // 随机选择起点

  Point start = { 

    rand() % (COLS / 2) * 2 + 1,

    rand() % (ROWS / 2) * 2 + 1 

  };

 maze->grid[start.y][start.x] = CELL_PATH;

 maze->start = start;

 vector<Point> walls;// 用于存储墙的位置

 int directions[4][2] = { {0, 1}, {1, 0}, {0, -1}, {-1, 0} }; // 上、右、下、左

 // 初始化墙列表

  for (auto& dir : directions) {

    int newX = start.x + dir[0];

    int newY = start.y + dir[1];

    if (newX > 0 && newX < COLS - 1 && newY > 0 && newY < ROWS - 1) {

      walls.push_back({ newX, newY });

    }

  }

 const int MAX_WALLS = (ROWS * COLS) / 2;// 最大处理的墙的数量

 int processedWalls = 0;// 处理的墙的数量

 // 随机生成迷宫

  while (!walls.empty() && processedWalls < MAX_WALLS) {

    // 随机选择一堵墙

    int index = rand() % walls.size();

    Point wall = walls[index];

    walls.erase(walls.begin() + index);

    processedWalls++;

    // 检查墙是否连接两个区域

    for (auto& dir : directions) {

      int cell1X = wall.x + dir[0];

      int cell1Y = wall.y + dir[1];

      int cell2X = wall.x - dir[0];

      int cell2Y = wall.y - dir[1];

      // 检查边界

      if (cell1X < 0 || cell1X >= COLS || cell1Y < 0 || cell1Y >= ROWS) continue;

      if (cell2X < 0 || cell2X >= COLS || cell2Y < 0 || cell2Y >= ROWS) continue;

      // 一个区域是路径，另一个是墙

      if (maze->grid[cell1Y][cell1X] == CELL_PATH && maze->grid[cell2Y][cell2X] == CELL_WALL) {

        // 打通墙和相邻的墙

        maze->grid[wall.y][wall.x] = CELL_PATH;

        maze->grid[cell2Y][cell2X] = CELL_PATH;

        // 添加新的墙

        for (auto& newDir : directions) {

          int newX = cell2X + newDir[0];

          int newY = cell2Y + newDir[1];

          if (newX > 0 && newX < COLS - 1 && newY > 0 && newY < ROWS - 1 &&

            maze->grid[newY][newX] == CELL_WALL) {

            walls.push_back({ newX, newY });

          }

        }

        break;

      }

    }

  }

  // 设置终点

 Point end ;

  if (rand() % 2 == 0) {

    // 在右边界

    end = { COLS - 2, rand() % (ROWS - 2) + 1 };

  }

  else {

    // 在下边界

    end = { rand() % (COLS - 2) + 1, ROWS - 2 };

  }

 maze->grid[end.y][end.x] = CELL_PATH;

 maze->end = end; 

 maze->grid[end.y][end.x] = CELL_END; 

}
```

### 5.2.3 路径搜索模块（BFS与DFS）

1. BFS算法

```C
PathNode* find_path_bfs(const Maze* maze) {

 // 队列和访问数组

 std::queue<Point> q;

 bool visited[ROWS][COLS];

 Point parent[ROWS][COLS]{};

 // 初始化访问数组和父节点数组

 memset(visited, false, sizeof(visited));

 for (int row = 0; row < ROWS; row++) {

  for(int col = 0; col < COLS; col++) {

   parent[row][col] = { -1, -1 };

  }

 }

 // 起点入队

 q.push(maze->start);

 visited[maze->start.y][maze->start.x] = true;

 // 方向数组：上、下、左、右

 int dx[] = { 0, 0, -1, 1 };

 int dy[] = { -1, 1, 0, 0 };

 while (!q.empty()){

  Point current = q.front();

  q.pop();

  // 如果到达终点，构建路径

  if (current.x == maze->end.x && current.y == maze->end.y) {

   PathNode* path = NULL;

   Point p = current;

   while (p.x != -1 && p.y != -1) {

    PathNode* node = new PathNode;

    node->pos = p;

    node->next = path;

    path = node;

    p = parent[p.y][p.x];

   }

   return path; // 返回找到的路径

  }

  // 遍历四个方向

  for (int i = 0; i < 4; i++) {

   int newX = current.x + dx[i];

   int newY = current.y + dy[i];

   // 检查新位置是否在迷宫内且未被访问

   if (newX >= 0 && newX < COLS && newY >= 0 && newY < ROWS && !visited[newY][newX] && maze->grid[newY][newX] != CELL_WALL) {

    visited[newY][newX] = true;

    parent[newY][newX] = current;

    q.push({ newX, newY });

   }

  }

 }

 return NULL; // 如果没有找到路径，返回NULL

}
```

```C
PathNode* find_path_dfs(const Maze* maze) {

 // 栈和访问数组

 std::stack<Point> s;

 bool visited[ROWS][COLS];

 Point parent[ROWS][COLS]{};

 //初始化访问数组和父节点数组

 memset(visited, false, sizeof(visited));

 for (int row = 0; row < ROWS; row++) {

  for (int col = 0; col < COLS; col++) {

   parent[row][col] = { -1,-1 };

  }

 }

 // 起点入栈

 s.push(maze->start);

 visited[maze->start.y][maze->start.x] = true;

 // 方向数组：上、下、左、右

 int dx[] = { 0, 0, -1, 1 };

 int dy[] = { -1, 1, 0, 0 };

 while (!s.empty()) {

  Point current = s.top();

  s.pop();

  //到达终点

  if (current.x == maze->end.x && current.y == maze->end.y) {

   // 回溯构造路径

   PathNode* path = NULL;

   Point p = current;

   while (p.x != -1 && p.y != -1) {

    PathNode* node = new PathNode;

    node->pos = p;

    node->next = path;

    path= node;

    p = parent[p.y][p.x];

   }

   return path; // 返回找到的路径

  }

  // 遍历四个方向

  for (int i = 0; i < 4; i++) {

   int newX = current.x + dx[i];

   int newY = current.y + dy[i];

   //检查新位置是否在迷宫内且未被访问

   if (newX >= 0 && newX < COLS && newY >= 0 && newY < ROWS && !visited[newY][newX] && maze->grid[newY][newX] != CELL_WALL) {

    visited[newY][newX] = true;

    parent[newY][newX] = current;

    s.push({ newX,newY });

   }

  }

 }

 return NULL; // 如果没有找到路径，返回NULL

}
```

## **6.3 问题与解决**

### 6.3.1 路径搜索内存泄漏

多次执行路径搜索后，程序内存占用持续增长，最终导致崩溃。原因是DFS递归实现中，回溯时未释放中间节点内存。BFS队列中节点出队后未及时删除。为解决此问题，在free_path函数中增加递归释放逻辑。

```C
void free_path(PathNode* path) {

 while (path) {

  PathNode* temp = path;

  path = path->next;

  delete temp;

 }
```

### 6.3.2 无法运行随机迷宫模式

点击按钮之后，虽然迷宫数据被更新了，但是界面没有正确刷新，进而看不到新生成的迷宫。在main.cpp中，每次循环都会清屏并重新绘制迷宫和相关元素并释放之前的路径，并将solution和current_pos指针置为NULL，避免出现指针错误。

### 6.3.3 读取访问权限冲突

`char direction=getDirection(old_pos->pos,current_pos->pos);`这一行代码抛出“读取访问权限冲突，`current_pos`是`nullptr`”异常，这表明在尝试访问`current_pos->pos`时，`current_pos`已经是`nullptr`了。发生在路径遍历到末尾，`current_pos`被置为`nullptr`的情况。在访问`current_pos->pos`之前，需要先检查`current_pos`是否为`nullptr`。如果`current_pos`为`nullptr`，则不应该尝试访问它的成员。为解决此问题，在原代码的基础上，添加了对`current_po`s是否为`nullptr`的检查：`if(current_pos!=old_pos&&old_pos&¤t_pos)`。

### 6.3.4 无法直接运行编译出的exe文件

在resource_manager.cpp中，资源文件使用相对路径resources/*.jpg，当在VS中运行时，工作目录是项目目录（包含resources文件夹），而当直接运行exe时，工作目录可能是exe所在目录（缺少resources文件夹），致使出现直接运行编译出的Maze.exe无法显示而调试器中可运行的问题。为解决此问题，将exe所在目录中增加resources文件夹。

# **附录：迷宫寻路系统编程程序说明书**

 

 

## **迷宫寻路系统编程程序说明书**

### **一、项目概述**

本项目是一个基于图形化界面的迷宫寻路系统，利用深度优先搜索（DFS）和广度优先搜索（BFS）算法来寻找迷宫中的路径。用户可以通过界面按钮生成固定迷宫或随机迷宫，并使用DFS或BFS算法进行寻路，同时支持对寻路结果进行测试。

### **二、功能特征**

1. 迷宫生成

固定迷宫：提供一个预设的迷宫布局。

随机迷宫：随机生成一个迷宫，起点和终点随机确定。

2. 寻路算法

BFS 寻路：使用广度优先搜索算法寻找从起点到终点的路径。

DFS 寻路：使用深度优先搜索算法寻找从起点到终点的路径。

3. 路径测试

可以对寻路结果进行动画演示，展示从起点到终点的移动过程。

4. 提示信息

在寻路成功或失败时，弹出提示框显示相应信息。

### **三、文件结构与功能**

1. main.cpp：程序的入口文件，负责初始化图形窗口、处理鼠标事件、绘制迷宫和控制按钮，以及循环处理用户交互。
2. resource_manager.h 和 resource_manager.cpp：负责加载迷宫所需的图像资源，如墙壁、路径、起点、终点等。
3. maze_defs.h：定义迷宫相关的数据结构和常量，如迷宫的行数、列数、单元格大小、单元格类型、点坐标、迷宫结构体和路径节点结构体等。
4. visualizer.h 和 visualizer.cpp：迷宫的可视化绘制，包括背景、迷宫、路径、控制按钮和测试结果提示框等。
5. maze_generator.h 和 maze_generator.cpp：生成固定迷宫和随机迷宫。
6. path_finder.h 和 path_finder.cpp：实现 BFS 和 DFS 寻路算法，并提供释放路径内存的函数。

### **四、使用流程**

1. 编译和运行

确保你的开发环境支持图形库（如 EasyX），将所有源文件编译链接后运行程序。

2. 界面操作

固定迷宫按钮：点击该按钮生成预设的固定迷宫。

随机迷宫按钮：点击该按钮随机生成一个迷宫。

BFS 寻路按钮：点击该按钮使用广度优先搜索算法寻找从起点到终点的路径。如果寻路失败，会弹出提示框显示 “无解迷宫！”。

DFS 寻路按钮：点击该按钮使用深度优先搜索算法寻找从起点到终点的路径。如果寻路失败，会弹出提示框显示 “无解迷宫！”。

开始测试按钮：在寻路成功后，点击该按钮开始动画演示，展示从起点到终点的移动过程。到达终点后，会弹出提示框显示 “测试成功！”。
